Как поднять проект-дор:

Deprecated: 1) Определить к какой группе доров относиться проект (с категориями или без)

1) Скопировать всё из проекта dor в проект с новым названием (например yurist123.ru - прям так с .ru и пишем)
2) Создать папку с названием домена в resources/projects и создать для домена все присущие ему сущности (.env.part, robots.txt, папку views) - все это скопировать из соседних папок и изменить все, что 	касается названия нового сайта
	- в файле .env.server_dor дописать две строки 
		DB_DATABASE=yurist123_ru (здесь название нового сайта)
		C_APP_HOST=yurist123.ru (здесь название нового сайта)
	- код счетчика кладем в resources/projects/новый_сайт/views/_partials/liveinternet.blade.php
3) В файле envoy прописать название домена (в переменную $project_name) (файл Envoy.blade.php)
	3а) в переменной @servers(['new_server'=>'127.0.0.1']) заменить 127.0.0.1 на IP-адрес удаленного сервера (192.81.223.20) (дроплета)

Создать новый репозиторий с таким же названием на Github и git push туда новый проект
	- скопировать SSHссылку нового репозитория и добавить в файл Envoy.blade.php в переменную $repo (например $repo = 'git@github.com:hvs-fasya/k-confiskat.ru.git';)
	 ВНИМАНИЕ еще раз для тех кто не заметил - SSH ссылку не HTTPS !!!!

4) Запустить envoy (поднимется проект: clone repo, composer, gulp, ...)
	- на локальной маштне в консоли в корневой директории проекта - команда envoy run deploy
	- в процессе деплоя все время, когда запрашивает пароль (root@192.81.223.20's password:), вбивать пароль от удаленного сервера

Если деплой прошел удачно желательно на GitHub ГРОХНУТЬ РЕПОЗИТОРИЙ этого проекта потому что он не приватный. Если клонирование удалось потом много всего и в конце было вот такое:

	Файл robots.txt был успешно скопирован
	Часть файла .env.part была успешно скопирована в .env файл
	root@192.81.223.20's password: 
	[192.81.223.20]: No migrations found.
	[192.81.223.20]: Migration table created successfully.
	[192.81.223.20]: Migrated: 2015_04_30_101426_create_contents_table
	Migrated: 2015_06_12_121154_update_contents_table
	Migrated: 2015_06_26_142305_add_indexes_contents_table
	Migrated: 2015_06_26_143459_add_kwtext_field

	значит скорее всего деплой удачный.

5) Добавить файл ключей для импорта в (storage/app), появившегося проекта на сервере через FTP/SSH или еще как-нибудь
	- файл ключей обязательно сохранить под названием keywords.txt - еще раз для тех кто в танке - НА СЕРВЕР!!! добавляем
	(если с локальной машины хотим добавить, то идем туда, где лежит файл и пишем 
6) Запустить команду импорта ключей через envoy (на локальной машине из корневой директории проекта)
	- envoy run import_keywords -> должно получиться что-то вроде "Было импортированно 29722 ключевых слов"
7) На сервере: (Через команду или вручную) Создать виртуальный хост для проекта (php artisan project:create_virtualhost --domain=yurist123.ru --ip=192.81.223.20)
	- команды php artisan .... запускаются на сервере из корня проекта
8) Повесить на крон консольные команды (парсинг ключей, перегенерация sitemap'а, ...)
	в cron (crontab -e на сервере) добавляется строка что-то вроде * * * * * php var/www/landings/yurist123.ru/artisan schedule:run 1>> /var/www/landings/yurist123.ru/parselog.txt 2>>/var/www/landings/yurist123.ru/parseerrors.txt
9) Подключить домен для проекта на сайте в панели доменов
	- на сайте digitalocean заходим в Networking там слева раздел Domains - там указываем имя и IP
	- не забыть делегировать домен там где раздают доменные имена
10) Не забывать никогда последним путктом на сервере в файле var/www/landings/yurist123.ru/.env установить APP_DEBUG=false
