Как поднять проект-дор:

Deprecated: 1) Определить к какой группе доров относиться проект (с категориями или без)
2) Создать папку с названием домена в resources/projects и создать для домена все присущие ему сущности (.env, robots.txt, views, ...) - все это скопировать из соседних папок
	- в robots.txt изменить название хоста
	- в .env изменить данные для БД: пользователя и его пароль
3) В файле envoy прописать название домена (в переменную $project_name) (файл Envoy.blade.php)
4) Запустить envoy (поднимется проект: clone repo, composer, gulp, ...)
	- envoy run deploy
5) Добавить файл ключей для импорта в (storage/app), появившегося проекта на сервере через FTP/SSH или еще как-нибудь
	- файл ключей обязательно сохранить под названием keywords.txt
6) Запустить команду импорта и опубликования ключей через envoy (run_keywords_commands)
	- envoy run run_keywords_commands
7) На сервере: (Через команду или вручную) Создать виртуальный хост для проекта (php artisan project:create_virtualhost --domain=otpravkimsk.ru --ip=195.154.62.136)
8) Повесить на крон консольные команды (парсинг ключей, перегенерация sitemap'а, ...)
	- парсинг -> php artisan content:parse
	- генерация файла сайтмэп -> php artisan sitemap:generate
9) Подключить домен для проекта на сайте в панели доменов.
